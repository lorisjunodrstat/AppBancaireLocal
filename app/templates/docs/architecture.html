Architecture & principes

Ce projet n’est pas seulement un outil. C’est une position technique et philosophique.

Local-first

L’application fonctionne entièrement en local. Aucune dépendance à un service externe n’est requise.

Vos données vivent sur votre machine ou votre serveur. Vous décidez où elles sont stockées, sauvegardées et exposées.

No lock-in

Aucun format propriétaire. Aucune API opaque. Aucun abonnement imposé.

Les données sont accessibles, exportables et compréhensibles. Vous pouvez quitter, modifier ou forker le projet à tout moment.

Auditabilité

Chaque calcul est explicable. Chaque écriture est traçable.

Il est possible de suivre une donnée depuis sa source (transaction, heure, événement) jusqu’à son impact comptable ou salarial.

Architecture modulaire

Chaque module est conçu pour fonctionner de manière autonome tout en s’intégrant dans une chaîne cohérente.

Banque → Comptabilité → Salaires → Reporting
